## План

### Фаза 0 — Инициализация проекта (30 мин)

- Создать Vite + React проект
- Настроить структуру папок
- Перенести текущий код из .jsx артефакта в компоненты
- Установить зависимости: socket.io, socket.io-client, express, tone, uuid
- Настроить dev-сервер с прокси для WebSocket

### Фаза 1 — Рефакторинг клиента (2-3 часа)

- Разбить монолитный .jsx на компоненты:
  - `Board.jsx` — доска (SVG или Canvas)
  - `Cell.jsx` — клетка
  - `PlayerPanel.jsx` — панель игроков
  - `ActionPanel.jsx` — кнопки действий
  - `DiceRoller.jsx` — кубики с анимацией
  - `Chat.jsx` — чат
  - `PropertyCard.jsx` — карточка владения
  - `AuctionModal.jsx` — модалка аукциона
  - `TradeModal.jsx` — модалка торговли
  - `VictoryScreen.jsx` — экран победы
  - `MainMenu.jsx` — главное меню
  - `GameSetup.jsx` — настройка партии
  - `Lobby.jsx` — НОВОЕ: лобби для онлайн
- Вынести игровую логику в `gameEngine.js` (shared между клиентом и сервером)
- Вынести данные в `gameData.js` (клетки, карточки, переводы)
- Добавить текстуры через CSS (фоны, паттерны, иконки)

### Фаза 2 — Сервер (2-3 часа)

- `server.js` — Express + Socket.io
- `roomManager.js` — управление комнатами
- `gameEngine.js` — общий движок (один код для клиента и сервера)
- `aiEngine.js` — ИИ логика (работает на сервере)

Серверные события Socket.io:
```
Клиент → Сервер:
  create_room    → создать комнату
  join_room      → присоединиться
  leave_room     → выйти
  start_game     → начать игру
  roll_dice      → бросить кубики
  buy_property   → купить
  start_auction  → начать аукцион
  auction_bid    → ставка аукциона
  end_turn       → завершить ход
  build          → строить
  mortgage       → залог
  redeem         → выкуп
  propose_trade  → предложить обмен
  accept_trade   → принять обмен
  decline_trade  → отклонить
  pay_jail       → заплатить за выход из тюрьмы
  use_card       → использовать карту свободы
  chat_message   → сообщение в чат
  lend_money     → одолжить при банкротстве
  skip_bankrupt  → пропустить голосование

Сервер → Клиент:
  room_created     → комната создана (roomId)
  room_joined      → игрок присоединился
  room_left        → игрок вышел
  game_state       → полное состояние игры
  game_update      → частичное обновление
  dice_rolled      → результат броска
  player_moved     → анимация движения
  property_bought  → покупка
  auction_update   → обновление аукциона
  card_drawn       → вытянута карточка
  chat_update      → новое сообщение
  turn_changed     → смена хода
  game_over        → конец игры
  error            → ошибка
```

### Фаза 3 — Онлайн-лобби (1-2 часа)

- Экран лобби: список комнат, создать/присоединиться
- Простая авторизация: имя + пароль (хранение в JSON)
- Комната: настройки игры, список игроков, кнопка "Начать"
- Добавить AI-слоты в комнату
- Reconnect при обрыве соединения

### Фаза 4 — Текстуры и ассеты (1-2 часа)

- Сгенерировать текстуры (пергамент, камень, дерево)
- SVG иконки для: домов, портов, утилит, карточек
- SVG гербы 6 домов (упрощённые)
- Фон главного меню — стилизованная карта Вестероса (SVG)
- Фишки игроков — SVG иконки
- Анимированные кубики (CSS 3D transform)

### Фаза 5 — Деплой (30 мин)

- Dockerfile для production
- Или: деплой на Railway / Render / VPS
- Переменные окружения: PORT, NODE_ENV
- Статическая сборка клиента в /dist
- Express раздаёт статику + WebSocket на том же порту

```
- Frontend: Vite + React

## ТЕКУЩАЯ СПЕЦИФИКАЦИЯ ИГРЫ

### Игровое поле
- 40 клеток (классический режим), 24 клетки (быстрый)
- 6 домов: Старки (серый), Ланнистеры (золотой), Таргариены (тёмно-красный), Баратеоны (тёмно-золотой), Тиреллы (зелёный), Грейджои (бирюзовый)
- Порты вместо вокзалов: Восточный Дозор, Браавос, Староместо, Белая Гавань
- Утилиты: Драконье пламя + Дикое огно
- Карточки: Вороны (шанс) + Железный трон (казна), по 16 штук
- Стена = тюрьма, Гнев Короля = "иди в тюрьму"
- Железный Трон = СТАРТ (бонус 200 драконов)
- Богороща = бесплатная парковка (ничего не происходит)
- Строительство: Башни (1-4) → Замок (5)
- Налоги: Налог Короны (200) + Дань Королю (100)
- Валюта: Золотые драконы

### Механики
- Аукцион по кругу при отказе от покупки
- Торговля: собственность + деньги + карты свободы
- До 3 даблов (на 3-й → тюрьма)
- Выход из тюрьмы: дубль / 50 драконов / карта свободы
- Голосование при банкротстве (другие могут одолжить)
- Залог / Выкуп (залог + 10%)
- Победа: последний небанкрот ИЛИ по кругам (выбор в настройках)
- Настраиваемые круги (10-50)

### AI
- 4 личности: Агрессивный, Осторожный, Торговец, Случайный
- 3 уровня сложности: Лёгкий, Средний, Сложный
- Пул имён: Серсея, Тирион, Дейнерис, Джон Сноу, Арья, Рамси, Джоффри, Тайвин, Санса, Петир, Варис, Бронн
- Скорость: 1-2 сек (имитация раздумья)

### UI/UX
- Двуязычие: RU/EN с переключением
- Тёмная тема GoT: чёрный (#0a0a12), тёмно-серый (#1a1a2e), золотой (#c9a84c), бордовый (#8B0000)
- Шрифт: Georgia (serif) для всего текста, letter-spacing для GoT стиля
- Фишки: 👑 Корона, ⚔️ Меч, 🐉 Дракон, 🏆 Кубок
- Доска на мобильном: уменьшенная копия с зумом
- Чат: попап по клику
- Панель игроков: сворачиваемая
- Анимация фишки: плавный перелёт
- Кубики: тёмные с золотыми точками, CSS 3D анимация
- Звуки: Tone.js (бросок кубика, движение фишки, покупка)
- Пауза (без сохранения)
- Карточка владения по клику: полная (цена, аренда, башни, залог)
- Экран победы: коронация со статистикой
- Главное меню: карта Вестероса (стилизация)
- Друзья: заглушка "Скоро!"
- Версия: "Kirshas Monopolia V1 © 2026"
```

```
## ВАЖНО

- Игровая логика ОДНА — gameEngine.js — используется и клиентом и сервером
- В локальном режиме gameEngine работает прямо в браузере (без сервера)
- В онлайн режиме клиент отправляет действия → сервер валидирует через gameEngine → рассылает результат всем
- AI работает на сервере в онлайн режиме, в браузере — в локальном
- Все SVG ассеты генерируй сам (не скачивай) — простые но стильные
```

```javascript
import express from 'express';
import { createServer } from 'http';
import { Server } from 'socket.io';
import { fileURLToPath } from 'url';
import { dirname, join } from 'path';

const __dirname = dirname(fileURLToPath(import.meta.url));
const app = express();
const server = createServer(app);
const io = new Server(server, { cors: { origin: '*' } });

// Production: serve static
if (process.env.NODE_ENV === 'production') {
  app.use(express.static(join(__dirname, '../dist')));
  app.get('*', (req, res) => res.sendFile(join(__dirname, '../dist/index.html')));
}

// Socket.io handlers
io.on('connection', (socket) => {
  console.log('Player connected:', socket.id);
  
  socket.on('create_room', (data) => { /* ... */ });
  socket.on('join_room', (data) => { /* ... */ });
  socket.on('roll_dice', (data) => { /* ... */ });
  // ... все остальные события
  
  socket.on('disconnect', () => {
    console.log('Player disconnected:', socket.id);
  });
});

const PORT = process.env.PORT || 3001;
server.listen(PORT, () => console.log(`Server on :${PORT}`));
```

- После каждой фазы проверяй что приложение запускается
- Если что-то сломалось — скажи конкретно что не работает
- Для текстур скажи: "Сгенерируй все SVG ассеты из плана"
- Для мультиплеера: "Реализуй фазу 2 и 3 — сервер и лобби"
- Для деплоя: "Настрой production build и Dockerfile"
